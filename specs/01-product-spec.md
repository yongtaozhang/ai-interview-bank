# 产品规格

## 产品定位

AI 前端面试宝典 MVP 是一个偏工具型、速查型的前端面试准备应用。它不是完整学习平台，也不是刷题系统，而是帮助用户快速检索、理解和复述资深前端面试题的轻量宝典。

## 用户画像

### 主要用户

- 有 8 年以上前端经验。
- 正在准备高级或资深前端岗位面试。
- 希望按知识方向快速复习。
- 希望答案更适合面试口述，而不是长篇教材式解释，并且抓住关键点
- 希望用该项目作为 AI Coding 实践作品展示。

## 用户路径

### 路径一：首页快速进入题库

1. 用户访问 `/`。
2. 看到项目定位、题库入口和推荐分类。
3. 点击进入 `/questions`。
4. 通过分类或高频标签查看题目。
5. 点击题目进入详情页。

### 路径二：关键词搜索

1. 用户访问 `/questions`。
2. 在搜索框输入关键词，例如 `闭包`、`React 性能`、`事件循环`。
3. 系统实时展示匹配题目。
4. 用户继续通过分类、难度或高频程度筛选。
5. 用户打开题目详情并能够复制口述答案。

### 路径三：查看 AI Coding 模板

1. 用户访问 `/ai-coding`。
2. 查看 Codex Prompt 模板。
3. 复制模板，用于后续需求拆解、代码生成或重构任务。

## 核心功能

### 首页

- 展示产品名称和一句话说明。
- 展示主要知识分类。
- 展示高频题入口。
- 展示进入题库和 AI Coding 页面的操作入口。

### 题库页

- 展示题目列表。
- 支持搜索题目标题、标签、关键点、口述答案。
- 支持按分类筛选。
- 支持按难度筛选。
- 支持按高频程度筛选。
- 展示当前筛选结果数量。
- 支持清空筛选。

### 题目详情页

- 展示题目标题。
- 展示分类、难度、高频程度、标签。
- 展示面试官考察点。
- 展示口述答案。
- 展示关键点列表。
- 展示常见追问。
- 展示延展阅读或关联题目。
- 提供复制口述答案按钮。

### AI Coding 页面

- 展示适用于 Codex 的 Prompt 模板。
- 模板按场景组织，例如需求拆解、组件实现、Bug 修复、重构、测试补齐。
- 提供复制模板能力。

## 信息优先级

1. 能否快速找到题目。
2. 能否快速判断题目是否重要。
3. 能否快速获得可口述答案。
4. 能否继续通过追问扩展复习。
5. 能否展示 Spec Coding 和 AI Coding 思路。

## 内容策略

第一版可以先内置一批高质量示例数据，覆盖以下方向：

- JavaScript 基础与进阶
- TypeScript
- React
- 浏览器与网络
- 前端工程化
- 性能优化
- CSS 与布局
- 前端安全
- 架构与项目经验
- AI Coding

## 非功能需求

- 首屏应轻量、快速。
- 搜索和筛选应即时反馈。
- 移动端可读可用。
- 所有题目详情应有明确层级。
- 复制成功应有明确反馈。

## MVP 风险

- 内容质量比功能数量更影响体验。
- 筛选项过多会让 MVP 复杂度上升。
- 若题目详情过长，需要良好的排版和锚点结构。
- 静态数据未来迁移到后端时，需要提前保持数据模型稳定。
