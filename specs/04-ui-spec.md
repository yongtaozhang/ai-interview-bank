# UI 规格

## 设计原则

- 工具优先：界面应服务快速查找和快速阅读。
- 层级清晰：标题、摘要、标签、答案、追问必须容易区分。
- 信息密度适中：面向资深前端复习，避免营销式大面积留白。
- 响应式：桌面端适合检索，移动端适合阅读。
- 风格克制：使用 Tailwind 构建干净、专业、可展示的作品感。

## 全局布局

### Header

- 左侧：产品名 `AI 前端面试宝典`
- 右侧导航：
  - 首页
  - 题库
  - AI Coding

桌面端横向导航，移动端可使用折叠菜单或简化导航。

### 页面宽度

- 主内容最大宽度建议：`max-w-6xl`
- 详情页正文最大宽度建议：`max-w-3xl` 到 `max-w-4xl`
- 横向内边距：移动端 `px-4`，桌面端 `px-6` 或 `px-8`

## 首页 `/`

### 页面目标

让用户快速理解项目用途，并进入题库或 AI Coding 页面。

### 信息结构

1. 首屏介绍区
   - 标题：AI 前端面试宝典
   - 副标题：前端资深面试题的速查、口述答案和追问清单
   - 主按钮：进入题库
   - 次按钮：查看 AI Coding 模板

2. 分类入口区
   - JavaScript
   - TypeScript
   - React
   - 浏览器与网络
   - 工程化
   - 性能优化
   - 架构与项目经验

3. 高频题提示区
   - 展示若干高频方向或高频题入口。

## 题库页 `/questions`

### 页面目标

支持用户通过搜索和筛选快速找到题目。

### 页面结构

1. 页面标题区
   - 标题：题库
   - 描述：按分类、难度和高频程度筛选前端面试题

2. 筛选区
   - 搜索框
   - 分类筛选
   - 难度筛选
   - 高频程度筛选
   - 清空筛选按钮

3. 结果摘要
   - 当前结果数量
   - 当前筛选条件提示

4. 题目列表
   - 卡片式列表
   - 每张卡片展示标题、摘要、分类、难度、高频程度、标签

5. 空状态
   - 当无结果时提示用户调整关键词或筛选条件。

## 题目卡片

### 必须展示

- 题目标题
- 简短摘要
- 分类 Badge
- 难度 Badge
- 高频程度 Badge
- 标签

### 交互

- 点击卡片进入 `/questions/:id`
- 键盘可聚焦
- Hover 和 Focus 状态明确

## 题目详情页 `/questions/:id`

### 页面目标

帮助用户快速理解题目、组织口述答案、准备追问。

### 页面结构

1. 返回题库入口
2. 题目标题
3. 元信息区
   - 分类
   - 难度
   - 高频程度
   - 标签
4. 面试官考察点
5. 口述答案
   - 复制按钮
   - 复制成功反馈
6. 关键点
7. 常见追问
8. 关联题目

### 排版要求

- 口述答案区域应比普通内容更突出。
- 关键点适合用列表。
- 追问适合用分组块。
- 不要让标签和按钮在移动端挤压换行异常。

## AI Coding 页面 `/ai-coding`

### 页面目标

展示可复用的 Codex Prompt 模板，用于项目展示和后续 AI Coding 工作流。

### 页面结构

1. 页面标题区
2. 模板分类导航或分组
3. Prompt 模板卡片
   - 标题
   - 适用场景
   - 简短说明
   - Prompt 内容
   - 复制按钮

## 视觉规范

### 字体

- 使用系统无衬线字体。
- 标题使用更高字重。
- 正文保证可读性，不使用过小字号。

### 颜色

建议使用中性色为主体，少量蓝色或青色作为强调色：

- 背景：浅灰或白色
- 正文：深灰
- 边框：浅灰
- 主操作：蓝色
- 成功反馈：绿色
- 高频标识：橙色或红色系，但保持克制

### 图标

使用 Lucide React：

- 搜索：`Search`
- 复制：`Copy`
- 成功：`Check`
- 返回：`ArrowLeft`
- 标签：`Tag`
- 筛选：`SlidersHorizontal`

## 响应式要求

- 移动端筛选区纵向排列。
- 桌面端筛选区可横向排列。
- 题目列表桌面端可使用两列或单列宽卡片。
- 详情页保持单列阅读体验。

## 交互反馈

- 搜索输入实时更新结果。
- 清空筛选后恢复全部题目。
- 复制成功显示短暂状态，例如按钮文字变为 `已复制`。
- 未找到题目时显示友好错误和返回入口。
