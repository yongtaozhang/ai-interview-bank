# Detailed Answer Strict Generation Spec

## Goal

`detailedAnswer` 必须生成专业、深入、结构清晰的面试解析答案。

它不是 `oralAnswer` 的扩写，不是口语回答，也不是简单八股总结。

`detailedAnswer` 应该像一段高质量技术文章中的核心解析，帮助用户真正理解问题背后的原理、机制、场景、边界和实践价值。

---

## Core Principle

`detailedAnswer` 必须严格遵守以下原则：

1. 先解释问题本质
2. 再解释机制来源或设计背景
3. 再拆解核心流程
4. 再分析关键细节
5. 再说明为什么这样设计
6. 再联系项目实践
7. 最后说明排查、优化、验证或常见误区

不能只写结论，必须解释原因。

---

简单来说……
